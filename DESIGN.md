# 설계

## 화면 설계

제시해주신 `llama.cpp`의 방대한 명령어 인수들을 기능별로 분류하여 정리해 드립니다. 이 설정값들은 모델의 성능, 메모리 사용량, 생성 결과의 품질을 결정하는 핵심 요소들입니다.

### 1. 하드웨어 및 성능 최적화 (Computing & Performance)

가장 중요한 섹션으로, CPU/GPU 자원을 어떻게 사용할지 결정합니다.

* **스레드 및 CPU 설정:**
  * `-t, --threads N`: 일반적인 텍스트 생성 시 사용할 CPU 스레드 수.
  * `-tb, --threads-batch N`: 프롬프트 분석(Batch) 시 사용할 스레드 수 (보통 `-t`보다 높게 설정하면 속도가 빠릅니다).
  * `-C, --cpu-mask`: 특정 CPU 코어에만 연산을 할당(Affinity)하는 고급 설정.
  * `--prio N`: 프로세스의 우선순위 설정 (실시간 또는 높은 순위 등).
* **GPU 가속 (CUDA/Metal 등):**
  * `-ngl, --n-gpu-layers N`: **가장 중요한 옵션.** GPU(VRAM)에 올릴 모델 레이어 개수입니다. `all`로 설정하면 성능이 극대화됩니다.
  * `-dev, --device`: 사용할 GPU 장치 선택.
  * `-sm, --split-mode`: 여러 개의 GPU가 있을 때 모델을 나누는 방식 (레이어별, 텐서별 등).
  * `-fa, --flash-attn`: 연산 속도를 높이고 메모리를 절약하는 Flash Attention 기법 활성화.

### 2. 모델 로딩 및 메모리 관리 (Model & Memory)

모델 파일을 불러오고 RAM/VRAM 효율을 관리합니다.

* **모델 경로:**
  * `-m, --model`: 불러올 모델 파일(`.gguf`)의 로컬 경로.
  * `-hf, --hf-repo`: 허깅페이스 레포지토리에서 직접 모델을 다운로드하며 실행.
* **메모리 최적화:**
  * `--mlock`: 모델 데이터가 스왑(하드디스크 사용)되지 않도록 RAM에 고정하여 속도 저하 방지.
  * `--mmap`: 메모리 맵핑 사용 여부 (대용량 모델 로딩 시 유리).
  * `-ctk, -ctv`: KV 캐시(문맥 유지 메모리)의 데이터 타입 설정 (f16, q8_0 등. 낮을수록 메모리 절약).

### 3. 문맥 및 생성 제어 (Context & Generation)

AI가 한 번에 기억할 정보량과 생성할 길이를 정합니다.

* `-c, --ctx-size N`: **문맥 크기.** AI가 한 번에 기억할 토큰 양입니다. (예: 4,096, 8,192)
* `-n, --n-predict N`: 생성할 최대 토큰 수. `-1`은 무한대입니다.
* `-b, --batch-size`: 한 번에 처리할 논리적 배치 크기.
* `-p, --prompt`: AI에게 전달할 첫 프롬프트 메시지.
* `-sys, --system-prompt`: AI의 페르소나를 정의하는 시스템 프롬프트.

### 4. 샘플링 파라미터 (Sampling - AI의 성격 결정)

AI가 단어를 선택하는 방식(창의성, 반복 등)을 조절합니다.

* `--temp, --temperature`: **창의성.** 수치가 높을수록 다양하고 예측 불가능한 답변을 하며, 낮을수록 결정론적이고 딱딱한 답변을 합니다.
* `--top-k, --top-p, --min-p`: 확률이 낮은 단어들을 걸러내는 필터링 기법들입니다.
* `--repeat-penalty`: 똑같은 말을 반복하지 못하게 방해하는 수치.
* `--mirostat`: 생성 품질을 일정하게 유지하기 위해 복잡도를 자동으로 조절하는 알고리즘.

### 5. 로깅 및 사용자 경험 (Logging & UI)

터미널에 표시되는 정보들을 제어합니다.

* `--perf`: 실행 후 성능 지표(토큰당 속도 등)를 출력.
* `--log-file`: 로그를 파일로 저장.
* `-co, --color`: 프롬프트와 답변을 색깔로 구분하여 가독성 향상.
* `--verbose`: 디버깅을 위해 아주 상세한 내부 동작 로그를 출력.
